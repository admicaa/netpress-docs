Basics

Controllers

Controllers stay thin in Netpress. They sit at the HTTP boundary, call services or models, and either return a value for the smart-response layer or use the response helpers that AppServiceProvider decorates onto res.

Quick Example

import { BaseController, json } from "@admicaa/netpress";
import RegisterRequest from "../../Requests/Auth/RegisterRequest.js";
import AuthService from "../../../Services/AuthService.js";

class AuthController extends BaseController {
  async register(req) {
    const payload = await this.validate(req, RegisterRequest);
    const result = await AuthService.register(payload);
    return json(result, 201);
  }
}

export default new AuthController();

BaseController

Extending BaseController gives you:

  • await this.validate(req, RequestClassOrRules)
  • await this.authorize(req, PolicyClass, method, resource?)

BaseController also binds inherited action methods to the controller instance, so passing PostController.show directly into the router works as expected.

Response Options

You can use the smart-response return style:

async show(req) {
  return Post.findOrFail(req.params.id);
}

Or the starter response helpers:

async show(req, res) {
  const post = await Post.find(req.params.id);
  if (!post) return res.error("Post not found", 404);
  return res.success(post);
}

AppServiceProvider decorates:

  • res.success(data, meta?)
  • res.error(message, status = 400, errors?)
  • res.paginate(items, total, page, perPage)
  • res.render(componentOrName, props, status = 200)

Smart Builders

Netpress also exports response builders when you want explicit intent:

import { json, redirect, view } from "@admicaa/netpress";

return json({ ok: true }, 201)
  .header("X-Resource", "post");

return redirect("/login")
  .withErrors({ email: ["Required"] })
  .withInput(req.body);

return view(HomePage, { title: "Dashboard" });

Those builders support shared methods like:

  • .status(code)
  • .header(name, value)
  • .withHeaders(map)
  • .cookie(name, value, options)

Controller Pattern

import { BaseController, json } from "@admicaa/netpress";
import Post from "../../Models/Post.js";
import StorePostRequest from "../Requests/StorePostRequest.js";

class PostController extends BaseController {
  async index(req) {
    const page = Number(req.query.page || 1);
    return Post.with("author")
      .where({ status: "published" })
      .latest()
      .paginate(page, 15);
  }

  async store(req) {
    const payload = await this.validate(req, StorePostRequest);
    const post = await Post.create({ ...payload, userId: req.user.id });
    return json(post, 201);
  }
}

This keeps the controller focused on HTTP concerns while the model and service layers own the data and domain behavior.
