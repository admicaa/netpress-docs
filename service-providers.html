Core Ideas

Service Providers

Service providers are the boot sequence of the starter. They register integrations, bind shared runtime services, and mount routes in a predictable order.

Quick Example

// bootstrap/providers.js
import AppServiceProvider from "../app/Providers/AppServiceProvider.js";
import DatabaseServiceProvider from "../app/Providers/DatabaseServiceProvider.js";
import AuthServiceProvider from "../app/Providers/AuthServiceProvider.js";
import MailServiceProvider from "../app/Providers/MailServiceProvider.js";
import EventServiceProvider from "../app/Providers/EventServiceProvider.js";
import RouteServiceProvider from "../app/Providers/RouteServiceProvider.js";

export default [
  AppServiceProvider,
  DatabaseServiceProvider,
  AuthServiceProvider,
  MailServiceProvider,
  EventServiceProvider,
  RouteServiceProvider,
];

What The Default Providers Do

ProviderResponsibility
AppServiceProviderglobal Express middleware, shared view renderer, shared flash state, response helpers
DatabaseServiceProviderboots Mongo or SQL runtime and registers shared application models
AuthServiceProviderauthentication services and strategies
MailServiceProvidermail transport and mail engine bindings
EventServiceProviderevent listener registration
RouteServiceProvidermounts web and API route trees

The Register Contract

Every provider exposes:

static async register(app) {
  // boot or bind services here
}

Providers run sequentially inside bootstrap/app.js, so they are the right place for app-wide setup that should happen once during boot.

A Real Route Provider Example

RouteServiceProvider now mounts web routes only when the configured web engine is enabled:

import { Router } from "@admicaa/netpress";
import apiRoutes from "../../routes/api.js";
import viewConfig from "../../config/view.js";

export default class RouteServiceProvider {
  static async register(app) {
    const router = Router();

    if (viewConfig.engine) {
      const { default: webRoutes } = await import("../../routes/web.js");
      router.use("/", webRoutes);
    }

    router.group("/api/v1", (api) => {
      api.use("/", apiRoutes);
    });

    app.use(router);
  }
}

A Custom Provider Example

import { container } from "@admicaa/netpress";
import Stripe from "stripe";

export default class PaymentServiceProvider {
  static async register() {
    container.bind("payments.stripe", () => {
      return new Stripe(process.env.STRIPE_SECRET_KEY);
    });
  }
}

Then add it to bootstrap/providers.js in the right order.

Guidelines

  • put global boot logic in providers, not route files
  • keep one provider focused on one area of responsibility
  • register dependencies before the providers that need them
  • mount routes late, after shared services are ready
