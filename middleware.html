Basics

Middleware

Middleware runs before the controller and handles cross-cutting rules like authentication, throttling, and authorization. It keeps route files readable and controllers focused.

Quick Example

// routes/auth.js
import { Router } from "@admicaa/netpress";

const router = Router();

router.group(throttle(10, 60), (router) => {
  router.post("/register", AuthController.register);
  router.post("/login", AuthController.login);
});

router.group(authMiddleware, (router) => {
  router.get("/me", AuthController.me);
});

Explanation

The starter kit already includes middleware for common needs:

  • auth.js verifies the JWT and loads req.user
  • role.js checks roles and policies
  • throttle.js limits repeated requests

Middleware should answer one question well. That makes route definitions easy to scan.

If your controller extends BaseController, you can move request validation into the action with this.validate(req, RequestClass) and keep only the transport rules in the route.

Additional Examples

Require an admin role for a whole group:

import { requireRole } from "../app/Http/Middleware/role.js";

router.group(authMiddleware, (router) => {
  router.group(requireRole("admin"), (router) => {
    router.get("/", UserController.index);
    router.delete("/:id", UserController.destroy);
  });
});

Use policy-based authorization:

import { authorize } from "../app/Http/Middleware/role.js";
import UserPolicy from "../app/Policies/UserPolicy.js";

router.delete(
  "/users/:id",
  authMiddleware,
  authorize(UserPolicy, "delete", async (req) => {
    return UserService.findById(req.params.id);
  }),
  UserController.destroy,
);

Advanced Usage

When you create a custom middleware, keep it stateless when possible. Read from req, decide, then call next() or throw an HTTP exception. If many routes need the same rule, wrap them in router.group() instead of repeating the middleware on every line.

Notes / Tips

  • Use middleware for access rules, not business workflows.
  • Throw exceptions from middleware and let the handler format the response.
  • Prefer small middleware factories over one large catch-all middleware.
