Security

Authentication

The starter kit includes token-based authentication out of the box. It covers registration, login, refresh, logout, and current-user endpoints.

Quick Example

// routes/auth.js
import { Router } from "@admicaa/netpress";

const router = Router();

router.group(throttle(10, 60), (router) => {
  router.post("/register", AuthController.register);
  router.post("/login", AuthController.login);
});

router.group(throttle(30, 60), (router) => {
  router.post("/refresh", AuthController.refresh);
});

router.group(authMiddleware, (router) => {
  router.post("/logout", AuthController.logout);
  router.get("/me", AuthController.me);
});

Explanation

AuthService handles the real work:

  • creating users
  • signing access and refresh tokens
  • rotating refresh tokens
  • blacklisting logged-out access tokens

auth.js middleware verifies the bearer token, loads the user from the database, and sets req.user.

AuthController can validate the register and login payloads inside the action with this.validate(...).

Additional Examples

Login request:

curl -X POST http://127.0.0.1:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@netpress.io","password":"password123"}'

Authenticated request:

curl http://127.0.0.1:3000/api/v1/auth/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Advanced Usage

The starter kit also includes optional Google and GitHub OAuth setup inside AuthServiceProvider. Keep email and password auth working first, then add OAuth when the app actually needs it.

Notes / Tips

  • Use throttling on login and registration routes.
  • Group auth routes when they share the same middleware.
  • Keep JWT secrets out of source control.
  • Treat req.user as the trusted authenticated user once authMiddleware passes.
