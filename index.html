Netpress Docs

Start Here

Netpress now starts from a published starter generator and a published framework package:

  • create-netpress-app scaffolds the application
  • @admicaa/netpress powers the runtime, CLI, and shared base classes

The starter is not just an API shell anymore. It includes shared models, unified migrations, shared seeders, optional server-rendered pages, React or Vue mail rendering, and a Laravel-style CLI on top of Express 5.

What You Get

  • Express 5 app bootstrapped through ordered service providers
  • API routes under /api/v1
  • optional web routes mounted from routes/web.js when config/view.js enables a web engine
  • shared app/Models/ classes that run against MongoDB, MySQL, PostgreSQL, or SQLite
  • unified database/migrations/ and database/seeders/ folders instead of separate SQL and Mongo trees
  • server-rendered web components in React or Vue
  • server-rendered mail components in React or Vue
  • the published @admicaa/netpress package underneath the starter app

Starter Flow

Create a new app with the generator, then finish the app-level setup:

npx create-netpress-app my-app
cd my-app
cp .env.example .env
npm run artisan -- key:generate
npm run setup:rendering
npm run migrate
npm run seed
npm run dev

That is the path the published starter kit now targets.

What Changed In The Starter

The biggest implementation shift is that the starter is now shared-first:

  • models live in app/Models/
  • migrations live in database/migrations/
  • seeders live in database/seeders/
  • driver-specific folders are now compatibility fallbacks, not the main structure

Netpress resolves the correct runtime per model and per connection, so application code does not need separate SQL and Mongo versions of the same model.

What To Keep In Mind

  • npx create-netpress-app installs dependencies for you unless you pass --skip-install
  • setup:rendering is part of normal setup, not an optional extra step
  • config/view.js controls whether starter web routes are mounted
  • config/mail.js always needs a mail engine of react or vue
  • the docs in this site are synced to the current starter kit and @admicaa/netpress
  • when you need roles, permissions, morph assignments, and package-level authorization middleware, use NetPress Permissions
